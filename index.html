Snapshots, Not Differences

Traditional Version Control System

Tend to store data as changes to a base version of each file

The Git Approach

Data as snapshots of the project over time

Sections of Git Project

  • Git Directory - The Git DB
  • Working Directory
  • Staging Area

Git Directory - The Git DB

Stores the metadata and object database for your project

Copied when you clone a repository

Working Directory

A single checkout of one version of the project; extracted from the db for modify

Staging Area

(AKA the index)

Stores information about what will go into your next commit

Workflow

  • Modify files in your working directory
  • Stage the files, adding snapshots of them to your staging area
  • Do a commit, stores the staged snapshot permanently to your Git directory
